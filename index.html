BAB 1 - PENDAHULUAN

1.1 Latar Belakang

Kompleksitas aplikasi web modern mengalami peningkatan signifikan dalam dekade terakhir, yang berdampak substansial terhadap performa dan pengalaman pengguna. Data dari HTTP Archive 2024 menunjukkan bahwa median JavaScript payload meningkat 14% dalam setahun terakhir, mencapai 558KB pada perangkat mobile dan 613KB pada desktop [1]. Lebih mengkhawatirkan lagi, penelitian empiris mengungkapkan bahwa sekitar 44% (206KB) dari JavaScript yang diunduh tidak digunakan selama proses pemuatan halaman, menunjukkan inefisiensi yang signifikan dalam optimasi payload [2].

Pertumbuhan ini bukan hanya fenomena statistik, melainkan memiliki dampak nyata terhadap Core Web Vitals dan metrik performa web. Data 2024 mengindikasikan bahwa hanya 49% website mobile dan 37% website desktop yang memenuhi standar Largest Contentful Paint (LCP) dengan waktu di bawah 2,5 detik [3]. Lebih kritis lagi, dengan diperkenalkannya Interaction to Next Paint (INP) sebagai pengganti First Input Delay (FID) pada Maret 2024, hanya 65% website mobile yang memberikan pengalaman interaksi yang baik dengan waktu respons di bawah 200ms [4].

üöÄ Framework JavaScript Modern dan Karakteristik Performa

Framework JavaScript modern seperti Vue.js dan React.js telah menjadi tulang punggung pengembangan aplikasi web single-page, namun masing-masing memiliki karakteristik performa yang berbeda. Penelitian komparatif oleh Emmanni (2023) [5] menunjukkan bahwa:

  • Vue.js mengungguli React.js dalam kecepatan manipulasi DOM hingga 758%
  • React.js menunjukkan keunggulan dalam time-to-interactive dengan waktu 300ms dibanding Vue.js yang mencapai 400ms
  • Analisis bundle size mengungkapkan aplikasi Vue.js yang dibangun dengan Vite menghasilkan payload sebesar 53,6KB, jauh lebih kecil dibanding React.js dengan create-react-app yang mencapai 531KB untuk aplikasi yang identik secara fungsional

üîß Peran Build Tools dalam Optimasi Performa

Build tools memainkan peran krusial dalam optimasi performa production, dengan perbedaan signifikan antara pendekatan tradisional dan modern [6]:

Build Tool Pendekatan Keunggulan Performa Build
Webpack Bundler matang Ekosistem plugin ekstensif, kemampuan optimasi teruji Baseline (100%)
Vite ESM-native esbuild pre-bundling, development yang cepat 20-30% lebih cepat

‚ö†Ô∏è Gap Penelitian dan Urgensi Akademis

Penelitian akademis tentang JavaScript bundling dan security implications oleh tim ACM SIGSAC (2023) [7] mengidentifikasi bahwa 40% website mengandung bundled code, namun analisis komprehensif tentang efektivitas berbagai build tools dalam mengurangi payload masih terbatas. Studi empiris menunjukkan bahwa penggunaan build tools yang tepat dapat mengurangi bundle size hingga 50% melalui teknik tree-shaking yang efektif [8], namun perbandingan sistematis antara Vite dan Webpack untuk framework Vue.js dan React.js belum dilakukan secara mendalam.

üíº Implikasi Bisnis dan ROI Performance

Pentingnya optimasi JavaScript payload tidak hanya terbatas pada aspek teknis, melainkan memiliki implikasi bisnis yang signifikan [9]:

  • Situs yang memuat dalam 1 detik memiliki tingkat konversi 3 kali lebih tinggi dibanding situs yang memuat dalam 5 detik
  • 50% konsumen meninggalkan keranjang belanja jika halaman tidak memuat dengan cukup cepat
  • Dengan demikian, pemilihan build tools yang optimal bukan hanya merupakan keputusan teknis, melainkan strategis yang mempengaruhi performa bisnis

üîß Variabel Independen (Faktor yang Dimanipulasi):
